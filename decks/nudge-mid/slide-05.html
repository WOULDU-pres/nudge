RALPH 내부 루프

Hypothesize → Plan → Act → Evaluate → Reason → Learn — 1회 시뮬레이션의 6단계

H
Hypothesize — 전략 가설 생성 비싼 모델 · AIDA 퍼널 구조의 전략 3개 생성
↓
P
Plan — 대상 선택 코드 · 200명 중 테스트 대상 페르소나 선택
↓
A
Act — 대화 시뮬레이션 싼 모델 × 2 · 전략 × 페르소나 조합으로 3턴 대화 (병렬)
↓
E
Evaluate — Judge 채점 비싼 모델 · 4차원 (engagement, relevance, persuasion, purchase_intent) + 퍼널
↓
R
Reason — 패턴 분석 비싼 모델 · 상위/하위 대화 비교 → 퍼널 병목 식별
↓
L
Learn — 학습 추출 비싼 모델 · 재사용 가능한 학습 포인트 → Ledger에 누적

💬 대화 구조 (A 단계)

턴 0: 세일즈 오프닝 Attention→Interest
턴 1: 가치 제시 Interest→Desire
턴 2: CTA Desire→Action
= 3턴 6메시지, AIDA 전체 커버

⚖️ 4차원 평가 (E 단계)

각 0-25점, 합계 0-100점
engagement relevance
persuasion purchase_intent
+ funnel_progress 0-3 (어디서 이탈?)

📚 Strategy Ledger

세대마다 누적되는 전략 원장.
proven_effective: 3세대 연속 성공 패턴
never_repeat: 3세대 연속 실패 패턴
→ 3-레이어 압축으로 context 폭발 방지

🔄 외부 루프와의 관계

RALPH = 실행자 ("전략 테스트")
Autoresearch = 진화자 ("프롬프트 자체 개선")
→ 메타 최적화
